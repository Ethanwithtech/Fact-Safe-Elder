# 老人短视频虚假信息检测系统 - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 项目背景
- **问题陈述**: 老人在抖音、微信号等平台刷短视频时，容易被金融和医疗诈骗虚假信息误导
- **目标用户**: 65岁以上老年人及其家属
- **核心价值**: 实时检测短视频中的虚假信息，保护老人免受诈骗

### 1.2 产品定位
一个基于AI的实时短视频虚假信息检测系统，通过浮窗形式为老人提供简单直观的风险提示。

### 1.3 技术创新点
- **多模态融合检测**: 结合文本、视觉、音频三种模态进行综合判断
- **跨模态注意力机制**: 基于FakeSV论文架构，实现模态间信息交互
- **实时处理**: 检测延迟<3秒，支持流式处理
- **可解释性**: 提供风险原因和建议措施

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 实时音频监听
- **功能描述**: 监听手机播放的短视频音频内容
- **技术要求**: 
  - 语音转文字 (STT) - 集成OpenAI Whisper
  - 支持中文方言识别
  - 实时处理，延迟<3秒

#### 2.1.2 文字内容抓取  
- **功能描述**: 获取短视频标题、字幕、评论等文字信息
- **技术要求**: 
  - OCR文字识别
  - 支持不同字体和背景
  - 抓取视频描述文本

#### 2.1.3 视觉内容分析 🆕
- **功能描述**: 分析视频画面中的可疑元素
- **技术要求**:
  - 使用ResNet50/EfficientNet提取视觉特征
  - 关键帧提取（首、中、尾）
  - 检测P图痕迹和AI换脸

#### 2.1.4 多模态虚假信息检测 🆕
- **功能描述**: 基于多模态AI模型判断内容是否为虚假信息
- **检测领域**: 
  - 金融诈骗 (投资理财、借贷、保险等)
  - 医疗虚假信息 (偏方、保健品、治疗方法等)
  - 假新闻和谣言
- **输出结果**: 
  - 安全 (绿色) - 置信度>0.7
  - 可疑 (黄色) - 置信度0.4-0.7
  - 高风险 (红色) - 置信度<0.4
- **模型架构**:
  - 文本编码器: BERT-base-chinese
  - 视觉编码器: ResNet50
  - 音频编码器: Whisper + BERT
  - 跨模态注意力融合

#### 2.1.5 智能浮窗提示
- **功能描述**: 在检测到风险时显示浮窗警告
- **设计要求**: 
  - 大图标，老人易识别
  - 颜色编码 (绿/黄/红)
  - 简单文字说明
  - 可自定义大小和位置
  - 语音播报功能 🆕

### 2.2 辅助功能

#### 2.2.1 家人提醒功能 🆕 增强
- **功能描述**: 当检测到高风险内容时，自动通知家人
- **通知方式**: 
  - 微信消息（模板消息）
  - 短信提醒（阿里云/腾讯云）
  - 邮件通知（SMTP）
  - App内推送
- **频率控制**: 
  - 每小时最多5条通知
  - 最小间隔5分钟
  - 可自定义阈值

#### 2.2.2 多级预警系统 🆕
- **功能描述**: 根据风险等级执行不同的预警措施
- **预警级别**:
  | 风险等级 | 颜色 | 动作 | 家人通知 | 内容阻断 |
  |---------|------|------|---------|---------|
  | 安全 | 绿色 | 正常显示 | 否 | 否 |
  | 可疑 | 黄色 | 温和提醒 | 可选 | 否 |
  | 高风险 | 红色 | 强烈警告 | 是 | 是 |

#### 2.2.3 历史记录
- **功能描述**: 记录检测历史，供家人查看
- **包含信息**: 
  - 检测时间
  - 风险等级
  - 内容摘要
  - 处理结果
  - 风险原因 🆕
  - AI模型解释 🆕

#### 2.2.4 设置管理
- **功能描述**: 允许调整检测敏感度和通知设置
- **设置项**: 
  - 检测敏感度（高/中/低）
  - 通知频率
  - 家人联系方式
  - 浮窗样式
  - 声音开关 🆕

## 3. 用户故事

### 3.1 主要用户场景

**场景1**: 老人刷抖音时遇到投资理财广告
- 用户: 李奶奶，67岁，在抖音看到"月入3万的理财秘诀"视频
- 系统行为: 
  1. 实时分析语音、文字和画面
  2. 多模态AI模型识别为金融诈骗风险（置信度92%）
  3. 浮窗显示红色警告
  4. 播放语音提示
  5. 自动发送微信消息给儿子
- 预期结果: 李奶奶看到警告，没有点击相关链接；儿子收到通知后打电话确认

**场景2**: 老人看到医疗保健品宣传
- 用户: 王大爷，72岁，观看"神奇保健品治百病"视频
- 系统行为: 
  1. 检测到"包治百病"、"祖传秘方"等高风险关键词
  2. AI模型判断为医疗虚假信息
  3. 显示黄色可疑提示
  4. 显示"有病请找正规医院"建议
- 预期结果: 王大爷意识到内容不可信，没有购买

**场景3**: 老人观看正常新闻内容
- 用户: 张阿姨，68岁，观看CCTV新闻视频
- 系统行为: 
  1. AI模型分析内容安全
  2. 浮窗显示绿色"内容安全"
  3. 3秒后自动隐藏
- 预期结果: 不打扰用户正常观看

```
│ • 管理界面      │    │ • ResNet视觉    │    │ • 模型存储      │
└─────────────────┘    │ • Whisper音频   │    └─────────────────┘
                       │ • 跨模态注意力   │
                       └─────────────────┘
                                │
                       ┌─────────────────┐
                       │   通知服务层     │
                       │                 │
                       │ • 微信通知      │
                       │ • 短信提醒      │
                       │ • 邮件推送      │
                       └─────────────────┘
```

### 4.2 AI模型架构 🆕
```
输入数据
  │
  ├── 文本 ──→ BERT-base-chinese ──→ 文本特征 [768维]
  │
  ├── 视频帧 ──→ ResNet50 ──→ 视觉特征 [768维]
  │
  └── 音频 ──→ Whisper ──→ 文本 ──→ BERT ──→ 音频特征 [768维]
                                            │
                            ┌───────────────┴───────────────┐
                            │      跨模态注意力融合层        │
                            │   (Multi-head Attention)      │
                            └───────────────┬───────────────┘
                                            │
                            ┌───────────────┴───────────────┐
                            │        分类头 [3类]            │
                            │  safe / warning / danger      │
                            └───────────────────────────────┘
```

### 4.3 性能要求
- **响应时间**: 检测延迟 < 3秒
- **准确率**: 虚假信息检测准确率 > 85%
- **召回率**: > 90%（优先不漏检）
- **F1分数**: > 87%
- **可用性**: 系统可用性 > 99%

### 4.4 安全要求
- **隐私保护**: 音频数据本地处理，不上传原始音频
- **数据加密**: 用户数据和通信加密
- **权限管理**: 细粒度的权限控制

## 5. 数据集和训练 🆕

### 5.1 支持的数据集
| 数据集 | 类型 | 语言 | 样本数 | 用途 |
|--------|------|------|--------|------|
| FakeSV | 短视频 | 中文 | 3,654 | 主要训练集 |
| Weibo21 | 图文 | 中文 | ~10,000 | 中文语境增强 |
| MCFEND | 新闻 | 中文 | ~5,000 | 多源验证 |
| 自定义 | 多模态 | 中文 | 可扩展 | 领域适配 |

### 5.2 训练方法
- **预训练**: 使用BERT-base-chinese和ResNet50预训练权重
- **微调策略**: 
  - 前5轮冻结编码器，只训练融合层
  - 后5轮解冻全部参数，小学习率微调
- **数据增强**: 文本同义词替换、图像翻转等
- **类别平衡**: 过采样少数类

### 5.3 训练环境
- **本地**: NVIDIA GPU (推荐RTX 3060+)
- **云端**: Google Colab (免费GPU)
- **框架**: PyTorch + Transformers
