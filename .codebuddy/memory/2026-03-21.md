## 第四阶段：修复 AI 检测 + 全屏风险警告 + OpenClaw 推送
### 修复：AI 实时检测结果不显示
- **根因**：`DetectionService.ts` 第50行 `{ ...response.data }` 解析错误
- 后端返回 `{ success, message, data: { level, score, ... } }`，真正的检测数据在 `response.data.data`
- 前端直接展开了外层 response.data，导致 level/score 等字段为 undefined
- **修复**：提取 `response.data.data` 为 apiData，正确映射到 DetectionResult
- **混合检测**：当 AI 模型判断为 safe 但本地关键词检测为 danger/warning 时，取更严格的结果（因为 AI 模型未训练好，随机权重，容易漏报）

### 新增：手机屏全屏风险警告覆盖层
- danger 内容显示红色全屏覆盖，带脉冲动画图标、风险原因列表、防骗提醒
- danger 需点击"我已知晓"按钮才能关闭
- warning 内容显示橙色覆盖，3秒后自动消失（点击也可关闭）
- safe 内容不显示覆盖

### 新增：OpenClaw 推送状态可视化
- 手机屏右上角显示推送状态小标签（发送中/已发送/失败）
- 全屏警告内显示"已通知家人（企业微信）"确认
- 右侧检测面板显示检测方法标签（AI/规则/混合）和 OpenClaw 推送标签
