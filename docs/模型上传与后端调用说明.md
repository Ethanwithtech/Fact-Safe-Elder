# 模型上传与后端调用说明

本文说明：**GitHub 上已包含哪些与 AI 相关的文件**、**它们是不是「模型参数」**、**后端 `enhanced_backend.py` 会调用谁**，以及**未上传文件里分别是什么内容**。

---

## 一、先回答两个关键问题

### 1. 上传的是「模型参数」吗？

| 类型 | 是否已上传到 GitHub | 说明 |
|------|---------------------|------|
| **模型结构 / 算法代码** | ✅ 是 | 例如 `models/bert_attention_detector.py` 里定义的 BERT + Attention + 分类头，这是**代码**，不是训练好的权重。 |
| **训练好的权重（真正的「参数」）** | ❌ 否 | 如 `simple_ai_model.joblib`、`fraud_detector_final.pth` 内含**大量数字参数**，体积大，**默认不上传**，需本地训练或 Release 分发。 |
| **指标与摘要（JSON）** | ✅ 是 | 如 `model_metrics.json`、部分 `training_result.json`，只记录**准确率、F1、路径说明**等小文本，**不是**神经网络权重。 |

结论：**仓库里主要是「会调用模型的代码」+「指标 JSON」；真正的 AI 权重文件需要你在本机生成或用网盘/Release 分享。**

### 2. 后端会调用上传的这些文件吗？

后端入口：**项目根目录的 `enhanced_backend.py`**（FastAPI）。

加载顺序（简化）：

1. **若存在** `models/fraud_detector_final.pth`  
   → 用 PyTorch 加载 **BERT + Attention** 结构（与 `bert_attention_detector.py` 思路一致），做三分类推理。  
   → 该 `.pth` **未上传**，需 Colab 训练后自行放入。

2. **否则若存在** `models/trained/simple_ai_model.joblib`  
   → 用 **sklearn + TF-IDF**（`train_real_model.py` 训练生成）做三分类。  
   → 该 `.joblib` **未上传**。

3. **若以上都不存在**  
   → 使用内置 **规则引擎**（关键词 + 正则），仍能返回 `safe` / `warning` / `danger`。

因此：**已上传的 `bert_attention_detector.py` 与仓库内文档，是「给开发者训练/对齐结构用的」；线上要跑「学习型模型」，必须在部署环境放置 `.joblib` 或 `.pth`。**

## 二、已上传到 GitHub 的 AI / 模型相关文件（清单与作用）

| 路径 | 作用 |
|------|------|
| `models/bert_attention_detector.py` | 论文思路下的 **PyTorch 模型类定义**（BERT、多头注意力、证据排序、分类头）。训练/推理脚本应对齐此结构；**不含**训练好的权重。 |
| `models/trained/model_metrics.json` | 某次本地训练后记录的 **准确率、F1、精确率、召回率** 及模型类型名；供文档与对比，**不是**模型参数文件。 |
| `models/finetuned/bert_mcfend/training_result.json` | 历史「模拟训练」脚本的 **结果摘要**（非真实大模型权重）。 |
| `models/README.md` | `models/` 目录说明与权重放置方式。 |
| `train_real_model.py` | 用 **sklearn** 在本地数据上训练，**生成** `simple_ai_model.joblib`。 |
| `download_real_training_data.py` | 生成/汇总 **训练用 JSON 数据**（见下文未上传部分）。 |
| `Colab_Training.py` | **Google Colab + GPU** 训练 BERT 类模型，**生成** `fraud_detector_final.pth` 的流程代码。 |
| `enhanced_backend.py` | **实际对外提供 `/detect` 等接口** 的后端，按上文顺序加载 `.pth` → `.joblib` → 规则。 |
| `start_system.py` | 启动后端 + 静态前端的辅助脚本。 |
| `docs/GITHUB_模型与AI上传说明.md` | 推送 Git 时的操作说明。 |
| `README.md`（根目录） | 项目总览与运行方式。 |
| `.gitignore` | 排除 `*.joblib`、`*.pth` 等大文件，避免误传。 |

**与旧后端 `backend/app/services/ai_models.py` 的关系**：若你仍用原 FastAPI 工程，那边可能加载 HuggingFace BERT；当前推荐的**可运行检测服务**以根目录 **`enhanced_backend.py`** 为准，并与上述 `models/` 路径一致。

## 三、未上传的主要文件：里面是什么？

以下文件被 `.gitignore` 排除或从未 `git add`，**不在 GitHub 仓库中**。

### 1. 模型权重（真正的「参数」）

| 文件（相对项目根） | 内容说明 |
|--------------------|----------|
| `models/trained/simple_ai_model.joblib` | **joblib 序列化对象**：内含 sklearn 的 **RandomForest（或同类）分类器** + **TfidfVectorizer**。即「整袋」模型，用于把中文文本变成向量再预测标签。体积通常 **数 MB～数十 MB**。 |
| `models/fraud_detector_final.pth` | **PyTorch 检查点**：通常含 `model_state_dict`（BERT+Attention 各层权重）、可选 `optimizer`、`training_info`。体积通常 **数百 MB 级**（取决于是否只存 state_dict）。 |

没有这两个文件时，后端仍可运行，但走的是 **规则引擎**，而不是学习型模型。

### 2. 训练数据（体积与版权）

| 路径 | 内容说明 |
|------|----------|
| `data/real_datasets/combined/*.json` | 合并后的 **train/val/test** 样本列表（文本 + 标签），由 `download_real_training_data.py` 生成；可能含 **LIAR 英文** + 自建中文样本。体积随样本量变化。 |
| `data/raw/mcfend/mcfend_data.json` | **MCFEND** 类多模态虚假新闻样本（常为大量 JSON 数组）。 |
| `data/raw/weibo_rumors/weibo_data.json` | 微博风格 **谣言/非谣言** 样本（项目内可能为占位或简化数据）。 |
| `data/raw/liar/*.tsv` | **LIAR** 数据集原始 **TSV**（政治声明真伪标签）。 |
| `data/raw/real_cases/real_case_dataset.json` | 整理的 **真实案例** 文本与标签。 |

这些数据**未默认上传**，多为体积与许可证考虑；需要时可单独用网盘或 **Git LFS** 管理。

### 3. 其它常见未跟踪文件

| 路径 | 内容说明 |
|------|----------|
| `keto-quiz-qrcode-*.png` | 与当前 AI 项目**无关**的二维码图片，**不应**与模型文档混在一起上传。 |
| `node_modules/` | 前端依赖，由 `npm install` 生成。 |
| `venv/`、`__pycache__/` | Python 虚拟环境与缓存。 |

## 四、部署到服务器时要带什么？

最少：

- 代码仓库（含 `enhanced_backend.py`、`models/` 下已提交的代码与 JSON）。
- 任选其一：**`simple_ai_model.joblib`** 或 **`fraud_detector_final.pth`**（按你想用的模型类型）。
- Python 依赖：`fastapi`、`uvicorn`、`torch`+`transformers`（若用 `.pth`）、`scikit-learn`+`joblib`（若用 `.joblib`）。

无权重时：仍可启动服务，接口返回基于 **规则** 的结果。
